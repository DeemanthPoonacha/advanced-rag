# Developer API Guide

This guide describes how to programmatically interact with the Advanced RAG Framework, implement custom plugins, handle streaming, and configure observability tracking.

---

## 🛠️ Orchestrator API Usage

The [RAGPipelineOrchestrator](../src/rag/pipeline/orchestrator.py) is the recommended entry point. It manages component construction, resource allocation, and lifecycle execution.

### Programmatic Ingestion and Query Execution

```python
import asyncio
from rag.config.loader import load_config
from rag.pipeline.orchestrator import RAGPipelineOrchestrator

async def run_pipeline():
    # 1. Load configuration
    config = load_config("config.yaml")
    
    # 2. Instantiate orchestrator
    orchestrator = RAGPipelineOrchestrator(config)
    
    try:
        # 3. Ingest documents (PDFs, Markdown, etc.)
        # Supports file paths or raw bytes
        chunk_ids = await orchestrator.ingest_source(
            source="docs/release_notes.txt",
            metadata={"product": "v3", "internal": True}
        )
        print(f"Ingested {len(chunk_ids)} chunks.")

        # 4. Ingest multiple documents concurrently in batches
        batch_ids = await orchestrator.ingest_batch(
            sources=["docs/page1.pdf", "docs/page2.pdf"],
            metadata=[{"page": 1}, {"page": 2}]
        )
        print(f"Ingested batch of {len(batch_ids)} chunks.")

        # 5. Query the engine
        # Evaluates safety guardrails and runs quality metrics in the background
        result = await orchestrator.query(
            user_query="What are the new release items in v3?",
            ground_truth="Release v3 contains performance updates.",
            metadata={"filters": {"product": "v3"}}
        )
        
        print(f"Answer:\n{result.answer}\n")
        print(f"Latency: {result.latency_ms:.2f} ms")
        
        # Access evaluation scores if configured
        if "evaluation" in result.metadata:
            metrics = result.metadata["evaluation"].get("metrics", {})
            print(f"Quality Metrics: {metrics}")

    finally:
        # 6. Release DB connections and client HTTP sessions
        await orchestrator.close()

if __name__ == "__main__":
    asyncio.run(run_pipeline())
```

## 🌊 Response Token Streaming

For interactive user interfaces, you can stream generated tokens live. Streaming runs input safety checks but bypasses output evaluation and metrics loops to minimize latency.

```python
import asyncio
from rag.config.loader import load_config
from rag.pipeline.orchestrator import RAGPipelineOrchestrator

async def stream_response():
    config = load_config("config.yaml")
    orchestrator = RAGPipelineOrchestrator(config)
    
    query = "How do I configure open telemetry?"
    print(f"Query: {query}\nResponse: ", end="", flush=True)
    
    try:
        # Iterate over tokens generated by the LLM
        async for token in orchestrator.query_stream(query):
            print(token, end="", flush=True)
        print()
    finally:
        await orchestrator.close()

if __name__ == "__main__":
    asyncio.run(stream_response())
```

## 🎨 Writing Custom Components

Every layer in the system uses Abstract Base Classes (ABCs). You can implement a custom component by extending the correct interface, using the `@ComponentRegistry.register` decorator, and updating your `config.yaml` with your provider key.

### Custom Chunker Example

```python
from typing import list
from rag.core.interfaces import BaseChunker
from rag.core.registry import ComponentRegistry
from rag.core.types import Chunk, Document

# Register your custom provider under the 'chunker' component type
@ComponentRegistry.register("chunker", "character_count_splitter")
class CharacterCountSplitter(BaseChunker):
    """Splits text into fixed character counts."""

    def __init__(self, limit: int = 1000, **kwargs):
        self._limit = limit

    async def chunk(self, document: Document) -> list[Chunk]:
        content = document.content
        chunks = []
        for i in range(0, len(content), self._limit):
            sub_text = content[i : i + self._limit]
            chunks.append(
                Chunk(
                    content=sub_text,
                    document_id=document.id,
                    metadata=document.metadata,
                    chunk_index=len(chunks),
                    token_count=len(sub_text.split())
                )
            )
        return chunks

    async def chunk_batch(self, documents: list[Document]) -> list[Chunk]:
        flat_list = []
        for doc in documents:
            flat_list.extend(await self.chunk(doc))
        return flat_list
```

To configure the pipeline to use this custom chunker, update `config.yaml`:
```yaml
ingestion:
  chunker:
    provider: "character_count_splitter" # Matches the registry registration name
    config:
      limit: 800
```

## 🔍 Distributed Tracing & Observability

You can track execution traces using the `@trace_operation` decorator and the `trace_span` context manager.

```python
from rag.core.types import LifecycleStage
from rag.observability.tracing import trace_operation, trace_span

# 1. Class method tracing decorator
@trace_operation(LifecycleStage.RETRIEVE, "custom_retriever_retrieve")
async def my_retrieval_func(self, query: str):
    # Method execution is traced as a span
    pass

# 2. Block-level tracing context manager
async def process_data(data: str):
    async with trace_span(LifecycleStage.CHUNK, "custom_parsing_block") as span:
        # Code run inside this block generates a nested trace span
        span.set_attribute("data_length", len(data))
        parsed = data.lower().split()
        return parsed
```

## 🌐 FastAPI REST API Endpoints

The FastAPI backend exposes the following endpoints for swappable database-driven document management, optimized for dynamic frontends:

### 1. Ingest Documents
* **URL**: `/api/ingest`
* **Method**: `POST`
* **Payload**: Multipart Form-data (`files` containing one or more files)
* **Response**:
```json
{
  "status": "success",
  "message": "Successfully ingested 1 files.",
  "files": [{"filename": "sample.pdf", "chunks_count": 12}],
  "total_chunks_ingested": 12,
  "chunk_ids": ["c1", "c2", ...]
}
```

### 2. List Ingested Documents
* **URL**: `/api/documents`
* **Method**: `GET`
* **Description**: Aggregates chunks from the vector database to dynamically list distinct uploaded documents, their chunk counts, token sizes, file types, and upload times.
* **Response**:
```json
{
  "status": "success",
  "documents": [
    {
      "name": "sample.pdf",
      "chunksCount": 12,
      "totalTokens": 3840,
      "file_type": "pdf",
      "uploadTime": "Jun 24, 2026, 12:45 PM",
      "status": "completed",
      "isMock": false
    }
  ]
}
```

### 3. Fetch Document Chunks (Lazy Loading)
* **URL**: `/api/documents/{filename}/chunks`
* **Method**: `GET`
* **Description**: Fetches individual chunk payloads (original text snippet, layout metadata, page numbers) for a specific document on-demand to optimize client rendering performance.
* **Response**:
```json
{
  "status": "success",
  "chunks": [
    {
      "id": "c1",
      "page": 1,
      "type": "text",
      "snippet": "Text segment...",
      "originalText": "Full text segment...",
      "summaryText": "",
      "isRaw": true,
      "metadata": { ... }
    }
  ]
}
```

### 4. Delete Ingested Document
* **URL**: `/api/documents/{filename}`
* **Method**: `DELETE`
* **Description**: Removes the document and all associated vector chunks from the active vector store dynamically.
* **Response**:
```json
{
  "status": "success",
  "message": "Successfully deleted document 'sample.pdf' from vector store."
}
```
